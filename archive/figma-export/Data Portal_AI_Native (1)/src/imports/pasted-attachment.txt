orders_by_city.sql — owner: A. Patel — created: 2026-03-01 — verified: true

SELECT city, COUNT(*) AS completed_orders
FROM orders
WHERE status = 'completed' AND order_placed_at >= DATEADD(day, -30, CURRENT_DATE)
GROUP BY city
ORDER BY completed_orders DESC;

delivery_latency_by_zone.sql — owner: M. Chen — created: 2026-02-27 — verified: false

SELECT zone_id, AVG(delivery_time_minutes) AS avg_latency
FROM deliveries
WHERE delivered_at >= DATEADD(day, -14, CURRENT_DATE)
GROUP BY zone_id
ORDER BY avg_latency DESC;

Screen Flows (step-by-step to prototype interactions)
Flow A — Agent → Pin → Auto Dashboard (demoable)

Open 02_DataAgent_Chat. Type question: “Why did completed orders drop in SF yesterday?”

Agent returns: one-line summary + 3 Result Cards (orders_by_city, delivery_latency_by_zone, courier_availability).

Hover a Result Card → reveal actions → click Pin. Repeat for second card. Toast: “2 cards pinned — Open canvas?” CTA → Open Canvas.

03_Draft_Dashboard_Canvas opens with pinned cards auto-laid out. Use right-rail to change card 2 chart type to “geo map”.

Click Publish → 04_Dashboard_Publish_Modal appears. Modal shows orders_by_city.sql (Verified) and delivery_latency_by_zone.sql (Unverified) with warning. Choose: Route delivery_latency_by_zone.sql to reviewer or publish as Draft. Click Publish (if draft, shows note).

Success toast: “Dashboard published. Share link copied.”

Flow B — Agent → Show SQL → SQL Studio → Promote

From Agent Result Card click Show SQL → overlay with SQL snippet (use orders_by_city.sql text).

Click Open in SQL Studio. 05_SQL_Studio loads with SQL prefilled.

Toggle Run on Sample and click Run → sample results table displays.

Click Promote to Metric → Promote Modal opens. Fill owner, add tag core:orders, click Create PR. Notification: “Metric draft created — assigned to A. Patel for verification.”

Flow C — Discovery → Clone → Notebook scaffold

01_Home_Discovery shows courier_availability_notebook.ipynb card (verified). Click Clone → 06_Notebook_Scaffold_Modal opens with scaffold options. Click Create Notebook to generate new notebook (demo as created state).

Flow D — Verification

Reviewer clicks notification: “Asset pending verification: delivery_latency_by_zone.sql.” Opens 07_Asset_Verification_Panel.

Panel shows checklist, PII scan (none), automated test results (performance: PASS). Reviewer checks items, clicks Approve. Asset badge changes to Verified and push to ingest/retrain queue.

Flow E — Telemetry

08_Telemetry_Ingest_Monitor shows recent ingest events, retrain queue length (2), last ingest time. Click latest ingest row to open log preview. Click Trigger Retrain to start manual retrain (show toast: “Retrain started — job id: r-20260310-01”).
