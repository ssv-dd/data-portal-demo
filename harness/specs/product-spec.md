# Product Specification: Dashboard Sharing, Chat-to-Dashboard Pinning, and Unified AI-BI Creation

## Overview

This specification covers three interconnected improvements to the Data Portal experience:

1. **Pin chart from home page chat to a dashboard** -- When the AI generates a chart as part of a chat response on the home page, users should be able to pin that chart directly to any of their dashboards, using the same destination picker already available in the dashboard creation flow.

2. **Read-only view for shared dashboards** -- Published dashboards should open in a read-only "report" view by default when accessed via the share URL. The creator view (editable canvas with left source panel, right AI sidebar, and editable title) should only appear when the user explicitly clicks "Edit." This makes dashboards feel like proper reports when shared with stakeholders.

3. **Consolidate dashboard creation by removing the right-side AI chat pane and enhancing the AI-BI source tab** -- The right-side AI assistant sidebar on the dashboard canvas page is redundant now that the left-panel already has an AI-BI chat tab. The right pane should be removed entirely. Instead, the AI-BI source tab should be enhanced with suggested prompts and the conversational look-and-feel currently found in the right sidebar. Users should still be able to add widgets directly (via the "Add Chart" button) or use the AI-BI chat to generate a chart and then pin it to the dashboard -- both paths should coexist.

## User Stories

- As a data analyst, I want to pin a chart generated in the home page chat to one of my dashboards so that I don't have to rebuild the same visualization manually in the dashboard editor.
- As a PM sharing a dashboard with my team, I want the shared URL to open in a clean, read-only report view so that viewers see a polished presentation rather than an editing interface.
- As a dashboard owner, I want to click "Edit" to enter the editing mode from the report view so that I can make changes when needed.
- As a dashboard creator, I want the AI-BI source tab to feel like a chat experience with suggested prompts so that I can use natural language to build charts without a separate right-side panel taking up screen space.
- As a user, I want the option to either add a widget directly (picking chart type and source manually) or use AI chat to create one, all from the same left panel, so that I have flexibility without redundant UI.

## Feature Requirements

### Feature 1: Pin Chart from Home Page Chat to Dashboard

- **Description:** When the AI generates a chart or analysis response within the home page chat interface, a "Pin to Dashboard" action should appear alongside the existing action buttons (Canvas, SQL, Notebook) in the chart section headers. Clicking it opens a dashboard picker that lets the user select an existing dashboard (or create a new one) and pins the chart as a new widget.

- **User Flow:**
  1. User submits a prompt in the home page chat box
  2. AI returns an analysis response that includes chart sections (e.g., the "Subscriber Growth Trend" area chart, the "Executive Summary by Region" table)
  3. Each chart/table section in the response shows a "Pin to Dashboard" button alongside existing action buttons
  4. User clicks "Pin to Dashboard"
  5. A modal appears listing the user's existing dashboards with search/filter, and an option to create a new dashboard
  6. User selects a dashboard and confirms
  7. The chart is added as a new widget on the selected dashboard
  8. A success confirmation (toast) appears with a link to view the dashboard

- **Acceptance Criteria:**
  - [ ] Each chart and table section in the analysis response displays a "Pin to Dashboard" action button
  - [ ] Clicking "Pin to Dashboard" opens a dashboard picker modal
  - [ ] The picker shows all user dashboards sorted by most recently edited, with search
  - [ ] The picker includes a "Create new dashboard" option
  - [ ] Selecting a dashboard and confirming adds the chart as a widget to that dashboard
  - [ ] A toast notification confirms success and provides a link to navigate to the dashboard
  - [ ] The newly pinned widget is highlighted briefly when the user navigates to the dashboard
  - [ ] Pin to Dashboard works for both chart visualizations and table sections

- **Edge Cases:**
  - User has no dashboards: the picker should show an empty state with a "Create new dashboard" prompt
  - Dashboard name is very long: truncate with ellipsis in the picker
  - Pinning while the target dashboard is open in another tab: the widget should appear on the next load/refresh

### Feature 2: Read-Only View for Shared Dashboards

- **Description:** When a user opens a published dashboard via the share URL (i.e., the URL generated by the Publish flow), the page should render in a read-only "report" mode. The left source panel, right AI assistant sidebar, editable title input, and editing controls (filter add, widget add/remove, drag handles) should all be hidden. An "Edit" button should be available (for the owner/authorized users) to switch into the existing editor view.

- **User Flow:**
  1. Dashboard owner publishes a dashboard and copies the share URL
  2. A viewer opens the share URL
  3. The page loads in report mode: full-width dashboard grid, read-only title, visible filters (non-editable), no side panels
  4. The top bar shows the dashboard title, domain/tier badges, and an "Edit" button
  5. Viewer can scroll, see all widgets, and interact with chart tooltips
  6. If the viewer clicks "Edit" (only visible to the owner or authorized editors), the page transitions to the full editor view with side panels and editing controls

- **Acceptance Criteria:**
  - [ ] When a dashboard URL contains the `shared` query parameter, the page opens in read-only report mode
  - [ ] In report mode, the left source panel is hidden
  - [ ] In report mode, the right AI assistant sidebar is hidden
  - [ ] In report mode, the title is displayed as plain text (not an editable input)
  - [ ] In report mode, widget drag handles, remove buttons, and edit buttons are hidden
  - [ ] In report mode, the "Add Chart" and "Publish" buttons are replaced with an "Edit" button
  - [ ] Dashboard-level filters are visible in report mode but not editable (no add/remove)
  - [ ] Clicking "Edit" switches to editor mode with all panels and controls restored
  - [ ] The canvas area uses the full page width in report mode (no side panel gaps)
  - [ ] Charts, tables, and KPI cards render identically in both modes

- **Edge Cases:**
  - Non-owner accessing the share URL: "Edit" button should not be shown (or show a permission message)
  - Dashboard with no widgets in report mode: show a meaningful empty state ("This dashboard has no widgets yet")
  - Browser back button after switching from report to edit mode: should return to report mode
  - Very long dashboard titles in report mode: truncate gracefully

### Feature 3: Consolidate AI-BI and Remove Right-Side Chat Pane

- **Description:** Remove the right-side AI assistant sidebar from the dashboard canvas page entirely. Enhance the existing AI-BI source tab in the left panel to incorporate the conversational feel of the removed sidebar -- specifically: suggested prompt chips, a welcoming initial state, and the same chat-like input style. The left panel's AI-BI tab becomes the single AI interaction point for dashboard creation. The "Add Chart" button in the top bar continues to work as a direct widget creation path (navigating to the chart builder), giving users both an AI-driven and a manual creation path.

- **User Flow (AI path):**
  1. User opens a dashboard in editor mode
  2. The left panel defaults to the AI-BI tab
  3. The AI-BI tab shows a welcome message and suggested prompt chips (e.g., "Show revenue by region," "Add a KPI for daily orders")
  4. User types a prompt or clicks a suggested chip
  5. AI processes the request and generates a chart preview within the left panel
  6. User confirms and the chart is added to the dashboard canvas

- **User Flow (Manual path):**
  1. User clicks "Add Chart" in the top bar
  2. User is taken to the chart builder page to configure a widget manually
  3. Upon completion, the widget is added to the dashboard

- **Acceptance Criteria:**
  - [ ] The right-side AI assistant sidebar is completely removed from the dashboard canvas page
  - [ ] The dashboard canvas area expands to fill the space formerly occupied by the right panel
  - [ ] The AI-BI tab in the left panel displays a welcome message and description when no conversation is active
  - [ ] The AI-BI tab shows suggested prompt chips that users can click to start building a chart
  - [ ] The AI-BI tab has a chat-style text input with send button at the bottom
  - [ ] The AI-BI tab retains its existing processing flow (thought steps, chart preview, add-to-dashboard confirmation)
  - [ ] The "Add Chart" button in the top bar still works and navigates to the chart builder
  - [ ] The maximize/minimize toggle works correctly now that only the left panel exists
  - [ ] Other left panel tabs (SQL, Semantic, Metrics, Cache) continue to work unchanged
  - [ ] The empty state on the dashboard canvas still directs users to the AI-BI tab

- **Edge Cases:**
  - Left panel collapsed: the AI-BI tab icon should still be accessible in the collapsed icon strip
  - Maximize mode: should collapse only the left panel (no right panel to worry about)
  - AI-BI tab conversation overflow: the chat area should scroll while the input stays fixed at bottom
  - Switching between left panel tabs mid-conversation: the AI-BI conversation state should persist

## UI/UX Structure

### Home Page Chat -- Pin to Dashboard

- The pin action button appears inline with existing section action buttons (Canvas, SQL, Notebook) in each chart/table section header of the analysis response
- The dashboard picker is a modal overlay with: search bar, scrollable list of dashboards (showing title, domain badge, widget count, last edited), and a "Create new dashboard" row at the top or bottom
- The toast notification on success appears at the bottom-right, with a "View Dashboard" link

### Dashboard Canvas -- Report Mode

- Report mode uses a single, full-width center panel with no side panels
- The top bar simplifies: static title text, domain/tier/published badges, "Edit" button on the right
- Filter bar shows existing filters as read-only chips (no add/remove controls)
- Widget cards render without drag handles, overflow menus, or remove icons
- Clean, presentation-grade appearance suitable for stakeholder viewing

### Dashboard Canvas -- Consolidated Left Panel AI-BI

- The left panel AI-BI tab replaces the right panel as the sole AI interface
- Initial state: sparkle icon, title ("AI-BI Assistant"), welcome text, and 4-6 suggested prompt chips
- Active state: scrollable chat messages area with input pinned to bottom
- The input area matches the existing AI-BI sidebar style: text area, send button, knowledge base selector
- Other source tabs (SQL, Semantic, Metrics, Cache) remain in the same left panel tab bar

## Data Model (Conceptual)

### Pin to Dashboard

- A chart generated in the home page chat needs to be translated into a `WidgetConfig` when pinned
- The chart type, title, and data from the analysis response map to widget properties
- The widget is persisted into the target dashboard's widget storage alongside a new layout entry
- No new persistent entities are needed; the existing Canvas/WidgetConfig/CanvasLayoutItem model is reused

### Report Mode

- The `shared` query parameter in the URL determines whether the dashboard opens in report mode
- No new data fields are required on the Canvas model; the `status: 'published'` flag plus the URL parameter drive the behavior
- Edit permission can be inferred from the `owner` field on the Canvas (compare with the current user)

### Consolidated AI-BI

- No data model changes; this is purely a UI restructuring
- Conversation state for the AI-BI tab is ephemeral (local component state), same as today
- Suggested prompts can be static or configurable per dashboard context

## Success Criteria

- [ ] A user can generate a chart in the home page chat and pin it to any existing dashboard without leaving the chat
- [ ] A shared dashboard URL opens in a clean, read-only report view by default
- [ ] The report view shows no editing controls and feels like a polished presentation
- [ ] An authorized user can switch from report view to editor view via a single "Edit" click
- [ ] The dashboard canvas editor has no right-side panel; all AI interaction happens through the enhanced AI-BI tab in the left panel
- [ ] Users can still add widgets either through AI chat (AI-BI tab) or manually (Add Chart button)
- [ ] The AI-BI tab feels conversational with suggested prompts, a welcome state, and chat-style input
- [ ] All existing dashboard functionality (drag-to-reorder, filters, publish, chart builder) continues to work

## Out of Scope

- Real-time collaboration or multi-user editing on dashboards
- Permission management UI (who can view vs. edit a shared dashboard)
- Backend API integration (all features use existing local storage / mock data patterns)
- Embedding dashboards in external tools (e.g., Slack, Confluence)
- Version history or undo/redo for dashboard edits
- Mobile-responsive layout for report mode
- AI conversation history persistence across sessions in the AI-BI tab
- Changes to the chart builder page itself
